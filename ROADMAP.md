# Wox Mail Daily Driver Roadmap

目标：成为可以长期日用、可替代 Thunderbird 的高性能低占用桌面邮箱客户端。

## 当前状态

Wox Mail 已经具备真实邮箱客户端的基础面：

- 多账户：Gmail、Outlook OAuth，以及自定义 IMAP/SMTP 账号。
- 收发信：IMAP 同步、SMTP 发送、回复、转发、本地草稿。
- 邮箱组织：远程文件夹列表、未读统计、标签、归档/垃圾邮件移动。
- 离线数据：SQLite 本地缓存、附件落库、Keyring/本机加密保存凭据。

这已经超过 UI 原型，但距离 Thunderbird 级别的日用替代，还缺少若干关键能力。

## P0: 日用可靠性

- IMAP 状态完整同步：同步远端已读/未读、删除、移动、UIDVALIDITY 变化，避免本地状态和服务器长期漂移。
- 删除/垃圾箱语义：提供删除、永久删除、从垃圾箱恢复，并按服务商特殊文件夹正确映射。
- 发送可靠性：失败重试、发件箱队列、断网后自动补发、Sent 文件夹远端追加。
- 附件发送：写信时添加附件，支持大附件大小提示和发送失败回滚。
- HTML 邮件安全阅读：支持安全渲染白名单 HTML，默认阻止远程图片和脚本，给用户手动加载远程内容。
- 账号错误恢复：OAuth 过期、授权码失效、服务器证书/网络错误要有明确状态和重新登录入口。

## P1: 高性能低占用

- 懒加载邮件体和附件：首轮同步只拉 Envelope、Flags、BodyStructure；打开邮件时再拉正文和附件。
- 分层缓存：正文、附件、缩略预览分开保存；附件可按需落盘，不默认把所有 BLOB 写进主数据库。
- 全文搜索索引：优先使用 SQLite FTS5；中文搜索可进一步引入 trigram 或专用分词索引。
- 后台同步调度：限制并发账号数和文件夹数，避免多账号同时同步造成网络和数据库尖峰。
- 数据库连接模型：读写连接分离，长同步任务避免阻塞 UI 读取。
- 缓存清理策略：按账号/文件夹限制本地保存天数、附件大小和总缓存占用。

## P2: Thunderbird 替代体验

- 规则和过滤器：按发件人、主题、收件人、关键词自动打标签/移动/标记。
- 统一收件箱和会话视图：跨账号合并、线程折叠、按 Message-ID/In-Reply-To 建立真实会话。
- 地址簿：联系人自动补全、常用联系人、头像、联系人分组。
- 快捷键：常用读信/写信/移动/搜索操作需要键盘可完成。
- 通知中心：新邮件通知应可点开对应邮件，按账号和文件夹配置。
- 导入迁移：从 Thunderbird/mbox/eml 或常见邮箱导入旧数据。
- 可访问性：焦点管理、屏幕阅读器标签、对比度和字号配置。

## 已开始改进

- 搜索命令已优先走 SQLite FTS5，并在 FTS 不可用或中文查询时回退到原有 LIKE 搜索。
- IMAP 和已发送 HTML 正文会先转为可读纯文本再进入当前阅读模型，避免阅读区出现原始 HTML 标签。
- IMAP 同步已记录 UIDVALIDITY；UIDVALIDITY 变化时会清理该文件夹本地缓存，避免使用失效 UID。
- IMAP 同步会定期刷新本地已缓存邮件的远端 `FLAGS UID`，同步远端已读/未读变化，并清理服务器上已删除的本地缓存邮件。
- 新同步的附件默认只保存元数据和附件顺序，不再把附件内容 BLOB 直接写进 SQLite；用户打开或保存附件时再从 IMAP 按需下载并缓存。
- 新同步邮件改为拉取 `BODY.PEEK[HEADER]` 和短正文预览，不再用 `RFC822` 整封下载；首次打开邮件时再用 `BODY.PEEK[]` 拉取完整正文并缓存，同时补齐附件元数据。
- 轻量同步已解析 `BODYSTRUCTURE`，列表页在未打开邮件前也能获得附件数量、文件名、类型和估算大小。
- 发送链路已加入本地 `outbox_jobs` 队列：写信先持久化，再尝试 SMTP 发送；失败会保留任务并按退避策略后台重试。
- SMTP 成功后会尽量通过 IMAP `APPEND` 写入远端 Sent 文件夹；APPEND 失败不会重复发送邮件，本地 Sent 仍会保留成功发送记录。

## 下一步建议

第一阶段先做“状态正确 + 占用下降”：

1. 增加可视化发件箱/失败队列视图，让用户能手动重试、取消或编辑未发送邮件。
2. 引入安全 HTML 渲染模型，区分 `body_text` 和 `body_html`，默认显示安全 HTML 或纯文本。
3. 为 IMAP 状态同步补充服务器移动场景识别和可观测错误状态。
4. 增加缓存策略设置，允许限制本地正文、附件和搜索索引占用。
